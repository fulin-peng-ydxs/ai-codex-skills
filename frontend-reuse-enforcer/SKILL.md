# Frontend Reuse Enforcer

前端开发默认复用优先。任何页面、组件、交互、状态、样式或数据处理模式第二次出现时，都必须先判断是否抽取或消费复用单元。

## 执行流程

1. 确定变更范围：
   - 用户指定页面、模块、文件、PR 或 diff 时，检查该范围及其影响到的同类实现。
   - 用户未指定范围但任务涉及前端开发时，从目标页面、组件目录、路由、设计文档、同类页面和近期改动入手。
2. 查找同类实现：
   - 使用 `rg` 搜索相似组件名、文案、class、函数名、状态字段、表格列名、筛选字段、API 参数、弹窗标题、空态文案和样式 token。
   - 读取至少一个高质量同类页面或组件，优先复用项目已有模式。
   - 若项目有 `DESIGN.md`、组件目录、页面模板、组件库、全局样式或工具库，先读取相关入口。
3. 做复用判定：
   - 若已有复用单元能覆盖，必须消费已有组件、hook/composable、util、API helper、schema、样式类或测试 helper。
   - 若已有一处相同模式，本次实现会成为第二处，必须优先抽取复用单元。
   - 若不抽取，必须给出理由，说明为什么不是同一变化原因、为什么抽象成本更高，或为什么当前需求不稳定。
4. 实施抽取：
   - 选择最低必要抽象层级：组件、hook/composable、util、schema、配置、样式、测试 helper。
   - 保持原有行为不变，先让旧使用点和新使用点都消费同一复用单元。
   - 抽象只承载共同变化点；差异通过明确 props、slots、options、callbacks、schema 或配置传入。
5. 验证闭环：
   - 跑项目要求的测试、类型检查、构建或相关单测。
   - 涉及页面 UI 时，必要时用浏览器或截图确认两个消费点都正常。
   - 如果项目有组件清单、DESIGN 文档或页面模板要求，按项目规则同步登记。

## 第二次出现规则

“第二次出现”不只指逐字复制。满足任一条件都触发复用判断：

- 相同结构：相似表格、表单、弹窗、抽屉、筛选栏、工具栏、卡片、空态、加载态、错误态、按钮组。
- 相同行为：相同打开/关闭、提交、重置、排序、分页、筛选、选择、批量操作、权限判断、状态切换。
- 相同数据处理：相同字段映射、格式化、参数归一化、枚举转换、状态文案、badge/tag 映射。
- 相同样式模式：相同布局 class、间距组合、颜色语义、圆角、阴影、响应式策略。
- 相同测试模式：相同 mock 数据构造、用户操作流程、断言 helper。

## 可抽取单元

- UI 结构和交互表现：抽组件。
- 组合状态、表单联动、权限判断、异步流程、布局计算：抽 hook/composable。
- 纯数据转换、格式化、字段映射、参数归一化：抽 util 或 API helper。
- 表格列、筛选项、表单字段、状态映射、操作按钮组：抽 schema/config/factory。
- 空态、加载态、错误态、权限态、提交中态：抽状态组件或状态渲染 helper。
- 重复样式组合：抽全局 class、CSS 变量、token、mixin 或组件级样式入口。
- 重复测试搭建：抽 test helper、mock builder、fixture factory。

## 允许不抽取的情况

不抽取必须在计划、实现说明或审查结论中说明理由。可接受理由包括：

- 只是极短、无领域含义的一两行表达，抽象后反而增加理解成本。
- 两处只是表面相似，领域语义、生命周期或未来变化方向明显不同。
- 当前需求还不稳定，抽象边界会过早固化错误模型；但应留下后续合并点。
- 复用会引入跨模块依赖、循环依赖或违反项目分层。
- 抽象会迫使调用方传入大量配置，复杂度超过重复本身。

不可接受理由：

- “先复制快一点”。
- “这次只改一个页面”，但同类实现已经存在。
- “稍后再抽”，但没有明确后续触发条件。
- “样式有一点不同”，但共同结构和行为明显一致。

## 必读清单

执行开发或审查时读取 `references/reuse-checklist.md`。

## 输出要求

前端开发计划或审查结论中包含：

```md
## 复用判断
- 已查找的同类实现：
- 已复用的组件 / hook / util / schema / 样式 / test helper：
- 本次新增的复用单元：
- 未抽取的重复点及理由：
- 需要同步维护的组件清单 / DESIGN / 测试：
```

## 边界

- 不为一次性、低风险、无稳定语义的极短代码强行抽象。
- 不创建万能组件、万能 hook 或巨大配置对象来追求表面复用。
- 不跨越项目分层、权限边界或模块所有权强行复用。
- 不因复用牺牲可读性；复用单元必须让两个以上调用点更清楚、更稳定或更易维护。
