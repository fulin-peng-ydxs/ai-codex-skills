# 更新策略

## 信息进入候选池的条件

满足任一条件即可进入候选池：

- 与 AI 推理、Agent、机器人、CPO、光互连、电力、散热、数据中心、资本开支相关；
- 不属于现有方向，但可能形成新的趋势、瓶颈或资本开支链条；
- 提到订单、产能、收入、利润、毛利率、交付周期、客户认证、技术路线；
- 可能验证或削弱现有方向研究；
- 来源可信但影响尚不明确；
- 来源较弱但话题可能重要。

## 信息进入正式文档的条件

至少满足一个条件：

- tier_1 或多个 tier_2 来源交叉验证；
- 直接影响趋势、瓶颈、资本开支、受益方、稀缺性、定价或验证信号；
- 改变文档的 `status` 或 `confidence`；
- 形成新增方向、新增案例或季度复盘结论。

## 不应直接进入正式文档的情况

- 只有社媒观点或无来源截图；
- 只有宏大叙事，没有资本开支传导；
- 只有股价波动，没有产业证据；
- 只有单一媒体转述，且原始来源不可追溯；
- 与现有知识库主题弱相关。

## 更新频率建议

| 类型 | 频率 | 动作 |
| --- | --- | --- |
| 候选池 | 每日或随时 | 追加信息，不改正式判断 |
| 方向研究 | 每周或重大事件 | 整合候选信息，更新判断链条 |
| 观察清单 | 每月 | 调整验证问题和优先级 |
| 复盘记录 | 每季度 | 记录验证、证伪和判断偏差 |
| 总纲 | 低频 | 只有方法论变化才更新 |

## 判断链条

更新正式文档时，优先回答：

```text
趋势是否变化？
瓶颈是否更清晰？
是否形成资本开支？
钱流向谁？
稀缺性是否增强或减弱？
市场是否已经充分定价？
有哪些验证信号？
```

## 自动化建议

自动化可自动更新正式文档，但必须遵守“先候选，后正式”的记录顺序。

推荐流程：

```text
定时收集公开信息
-> 根据 active_research_period 或当前日期选择候选池/观察清单/复盘文件；active_research_period 默认与当前自然季度一致
-> 追加候选池
-> 根据来源层级和判断链条自动决定是否更新正式文档
-> 跑校验脚本
```

正式文档自动更新的约束：

- 明确来源白名单；
- 明确影响方向；
- 保留候选记录；
- 保留更新记录；
- 校验通过后才报告成功；
- 冲突、证据不足或重大方法论变化时标记 `review`。

## 网络受限策略

网络受限时，自动化仍应完成本地闭环：

```text
识别网络失败
-> 不编造外部事实
-> 记录 metadata/update-state.json 的 last_status 和 last_error
-> 继续处理本地候选池和已有文档
-> 对需要外部确认的事项标记 review
-> 运行本地校验
-> 报告网络受限和已完成的本地处理
```

日频任务如果完全无法访问外部来源，只更新 `metadata/update-state.json`，不创建空候选记录。周频、月频、季度任务可以继续使用本地已有候选信息和正式文档。
